08 — Components

Buttons

Buttons carry intent. A clear visual hierarchy — primary to ghost — ensures users always know what matters most. One primary action per view.

Variants
Sizes
With Icons
States
✓ Do
  • One primary button per view section
  • Use action verbs — "Save Changes", "Create Account"
  • Keep labels concise — 1 to 3 words max
  • Always explain why a button is disabled
✕ Don't
  • Multiple primary buttons in the same area
  • Vague labels — "Click Here", "Submit", "OK"
  • ALL CAPS beyond extra-small size
  • Disable without showing a reason or tooltip

16 — Interaction

Voice & Tone

What we say matters as much as how it looks. Our writing is confident, warm, and precise — never corporate, never cute.

🧠
Clear
Say exactly what's happening. No jargon, no fluff. Users should never have to guess.
🤝
Human
Write like a knowledgeable friend, not a manual. Contractions are fine. Empathy is required.
⚡
Direct
Front-load the action. Lead with a verb. Keep button labels to 1–3 words. Always.
✓ Write this
  • "Your changes have been saved."
  • "Something went wrong. Try again."
  • "Delete this project?"
  • "You're all caught up 🎉"
  • "Upgrade to unlock this feature."
✕ Not this
  • "Your request has been successfully processed."
  • "An unexpected error has occurred."
  • "Are you absolutely sure you want to proceed?"
  • "No new notifications at this time."
  • "This feature is unavailable on your plan."
17 — Rules

Usage Rules

Non-negotiables that keep the system coherent. These aren't suggestions — they're the guardrails that make everything feel intentional.

📐
Always use tokens, never raw values
Never write #2563eb in a component. Use var(--color-accent). This ensures system-wide updates happen in one place.
🔡
Respect the type hierarchy
One h1 per page. Never skip levels. Don't use bold to add emphasis — use hierarchy instead.
🎯
One primary action per screen
If everything is important, nothing is. The primary button is a focal point — it should guide users toward their most valuable next step.
🪶
Whitespace is not wasted space
Breathing room signals confidence. Tight, crowded layouts feel anxious. Trust the spacing scale — never compress gaps to fit more content.
♿
Accessibility is a feature, not a checklist
Every interactive element needs a visible focus state. Every icon-only button needs an aria-label. Test with a keyboard — every time.
🔄
Animate with purpose
Motion should explain, not entertain. If removing an animation doesn't break understanding, remove it. Always respect prefers-reduced-motion.
Great systems don't constrain
creativity — they focus it.

When in doubt, return to purpose. Does this serve the user? If yes — ship it. If not — simplify it.
